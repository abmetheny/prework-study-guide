HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • Three ways to style a webpage:
    • Inline CSS: Applies style to a specific tag directly in the HTML file.
    • Internal Style Sheet: Embeds stylesheet in HTML file using a <style> element in the <head> element. Rules only apply to that page.
    • External Style Sheet (most common): CSS rules are kept in a separate file, using a <link> element in <head>.
  • CSS Rules contain Selectors (the element/attributes to which the rule applies) and Declarations (the CSS property to apply and the value of the property)
  • Using * as Selector applies the rule to all visible elements on a page.
  • Margin is the space around the outside of an element; Padding is the space around the content inside an element.

Git

  • git status: checks what branch you are currently on
  • git checkout -b [branch-name]: creates a new branch and switches to it
  • git add -A: stages files (always stage after saving changes)
  • git commit -m "[commit message]": leave a detailed message about what changedn
  • git pull origin main: pulls main source code to ensure feature branch is up to date with main before merging
  • git push origin [name of your feature branch]: merges feature branch changes into main

JavaScript

  • JavaScript is case sensitive.
  • Whereas CSS styles are linked within the <head> element,the JS file is linked within the <body> element.
  • Variables: Containers that store values
    • Declare a variable: let [myVariable];
    • Give the variable a value: let [myVariable] = "[value]"
    • Retrieve the value: [myVariable];
    • Change the value later in the code: [myVariable] = "[newValue]"
    • Data types: String, Number, Boolean (true/false), Array (multiple values), Object (anything else)
  • Method: A set of instructions that can be executed by a computer.
  • Add a comment: // [comment] or /* [comments spanning multiple lines] */
  • Operator: Mathematical symbol based on two values/variables (Add, Subtract, Multiply, Divide; Assignment; Strict Equality; Not, Does-Not-Equal).
  • Conditionals: Tests if an expression returns true or not (e.g., if...else, while).
  • Functions: A set of instructions that tells the computer how to perform a certain task; must be called by name in order to execute. Looks like a variable name followed by (), then add {} that contain the code for the task we want to perform.
  • Events: Code structures that listen for activity in the browser and run code in response.
  • Null: Referse to the absence of a value.
  • Objects are the only mutable values in JavaScript. The two types of object properties, data (value, writable, enumeable, configurable) and accessor (get, set, enumberable, configurable), each have corresponding attributes.
  • JavaScript is weakly typed; you can often use a value of one type where another is expected and it will be converted to the right type for you.
  • Control Flow: The order in which a computer executes code in a script. In JavaScript, from the first statement at the top of the page down to the last statement at the bottom of the page.
  • Falsy values include 0, -0, On, "", null, undevined, NaN, or a Boolean data type with a false value. All other valudes (including Booleans with a true value) are considered truthy.
  • For including multiple conditions, use if...else if.
  • Arrays store groups of variables and are not limited to a single type of data. Each item is assigned a unique number (i.e., index) that can be used to access individual data inside the array, usually starting with 0 and increasing by one whole number each time.
  • Strings must be contained within quotation marks.
  • A for loop performs a task on all items in an array and contains three important statements:
    • Starting Point (var x = 0)
    • Condition (x < shapes.length;)
    • Iterative Pattern (x++, which is a shortcut for x+1)
